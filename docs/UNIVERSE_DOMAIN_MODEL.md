# Доменная модель: Вселенная гиперразумных существ

## Описание

Доменная модель основана на тексте из "Автостопа по галактике" Дугласа Адамса. Раса гиперразумных всемерных существ устала от постоянных споров о смысле жизни и решила построить суперкомпьютер для решения этого вопроса.

## Основные классы

### 1. **Universe** - Вселенная
Центральный класс, управляющий всей системой.

**Ответственности:**
- Управление планетами
- Создание и управление суперкомпьютером
- Отслеживание истории событий
- Проведение глобальных турниров по брокианскому ультра-крикету

**Основные методы:**
- `addPlanet(planet: Planet)` - добавить планету
- `buildSupercomputer(name: String)` - построить суперкомпьютер
- `startSearchingForAnswer()` - начать поиск ответа
- `passTime(timeUnits: Long)` - симуляция времени
- `isTiredOfDebates()` - проверка усталости от споров

### 2. **Planet** - Планета
Место обитания существ.

**Атрибуты:**
- `name: String` - название
- `mass: Double` - масса
- `inhabitants: MutableList<Habitant>` - обитатели

**Основные методы:**
- `addInhabitant(habitant: Habitant)` - добавить обитателя
- `conductUltraCricketTournament()` - провести турнир
- `getAverageFrustrationLevel()` - средняя степень разочарования

### 3. **Habitant** - Обитатель
Существо, населяющее вселенную.

**Атрибуты:**
- `name: String` - имя
- `race: Race` - раса
- `currentActivity: TypeOfActivity` - текущая деятельность
- `frustrationLevel: Int` - уровень разочарования (0-100+)
- `knowledgeLevel: Int` - уровень знаний

**Основные методы:**
- `startDebating()` - начать спорить (повышает frustration)
- `playBrockianUltraCricket(target: Habitant)` - играть в крикет (снижает frustration)
- `rest()` - отдохнуть (снижает frustration)
- `contemplate()` - размышлять (повышает knowledge)
- `canBuildSupercomputer()` - может ли построить суперкомпьютер

### 4. **Supercomputer** - Суперкомпьютер
Компьютер для вычисления ответа на главный вопрос.

**Атрибуты:**
- `name: String` - название (например, "Deep Thought")

**Основные методы:**
- `startCalculation()` - запустить вычисление
- `tick(timeUnits: Int)` - продвинуть вычисление
- `getAnswer()` - получить ответ (42)
- `askQuestion(question: String)` - задать вопрос

**Особенности:**
- Вычисление занимает 7.5 млн условных единиц времени
- Ответ всегда равен 42

### 5. **Debate** - Дискуссия
Спор о смысле жизни.

**Основные методы:**
- `addParticipant(habitant: Habitant)` - добавить участника
- `start()` - начать дискуссию
- `conductRound()` - провести раунд
- `end()` - завершить дискуссию
- `needsDrasticMeasures()` - нужны ли кардинальные меры

### 6. **Race** (Enum) - Расы
- `HYPERINTELLIGENT_PANDIMENSIONAL_BEINGS` - гиперразумные всемерные существа
- `HUMAN` - люди
- `MANDALORIAN` - мандалорцы
- `VOGON` - вогоны
- `DOLPHIN` - дельфины

### 7. **TypeOfActivity** (Enum) - Виды деятельности
- `DEBATING_MEANING_OF_LIFE` - спор о смысле жизни
- `PLAYING_BROCKIAN_ULTRA_CRICKET` - игра в крикет
- `CONTEMPLATING` - размышление
- `RESTING` - отдых
- `BUILDING_SUPERCOMPUTER` - строительство суперкомпьютера
- `WAITING_FOR_ANSWER` - ожидание ответа

## Взаимодействия и сценарии

### Сценарий 1: Полный жизненный цикл
1. Создается вселенная с планетами
2. На планеты добавляются гиперразумные существа
3. Начинается дискуссия о смысле жизни
4. Дискуссия продолжается много раундов
5. Существа устают (frustrationLevel > 50)
6. Принимается решение построить суперкомпьютер
7. Суперкомпьютер запускает вычисление
8. Проходит 7.5 млн единиц времени
9. Получен ответ: 42

### Сценарий 2: Брокианский ультра-крикет
1. Существа спорят и накапливают frustration
2. Проводится турнир по брокианскому ультра-крикету
3. Существа бьют друг друга и убегают
4. Activity меняется на PLAYING_BROCKIAN_ULTRA_CRICKET
5. Frustration немного снижается

### Сценарий 3: Развитие существа
1. Существо размышляет (contemplate) - растет knowledgeLevel
2. При достижении knowledgeLevel >= 10 и наличии нужной расы
3. Существо может участвовать в создании суперкомпьютера

## Метрики и состояния

### Уровень разочарования (Frustration)
- 0-30: Нормальное состояние
- 31-50: Повышенное раздражение
- 51+: Усталость от споров (isTiredOfDebates)

### Уровень знаний (Knowledge)
- 0-9: Недостаточно для создания суперкомпьютера
- 10+: Может участвовать в создании (для гиперразумных существ)

## Особенности реализации

1. **Изменяемое состояние**: Объекты имеют изменяемые свойства для симуляции динамики
2. **Побочные эффекты**: Методы могут изменять состояние других объектов
3. **История событий**: Universe логирует все важные события
4. **Временная симуляция**: Метод `passTime()` позволяет двигать время вперед
5. **Условия и ограничения**: 
   - Только гиперразумные существа с knowledge >= 10 могут строить суперкомпьютер
   - Для турнира нужно минимум 2 участника
   - Суперкомпьютер можно построить только один раз

## Расширяемость

Модель легко расширяется:
- Добавление новых рас
- Новые виды деятельности
- Дополнительные метрики (здоровье, счастье и т.д.)
- Новые типы событий
- Взаимодействие между планетами
